# Manager Checklist -- Week 1

Setup tasks for the manager before interns start.

## Before Interns Arrive

- [ ] Obsidian vault in a Git repo (use `vault-template/` as starting point)
- [ ] Folder structure created: per-intern folders + `Shared/` + `Company/`
- [ ] Onboarding guide written in `Company/Onboarding/`
- [ ] PR and commit conventions documented
- [ ] Weekly 30-minute review slot scheduled per intern
- [ ] Claude Code API keys provisioned for each intern
- [ ] Project repos accessible to interns

## Day 1

- [ ] Walk interns through ATLAS framework (use `index.html` slides)
- [ ] Verify each intern has tools installed (see `TOOLS.md`)
- [ ] Each intern creates their personal vault folder
- [ ] Demonstrate CRISPY methodology with a simple example
- [ ] Set expectations: daily session logs, daily git push, CRISPY before code

## During Week 1

- [ ] Review first session logs -- are they being written daily?
- [ ] Check git activity -- are commits descriptive and frequent?
- [ ] Verify CRISPY is being used -- are artifacts appearing in `.crispy/`?
- [ ] First weekly review at end of week 1

## Ongoing Oversight Cadence

| Frequency | Activity |
|-----------|----------|
| Daily (async) | Glance at git pushes and session logs |
| Weekly (30 min) | Review git activity, walk a CRISPY artifact, promote research to `Shared/` |
| Monthly | Cohort retrospective, promote patterns to `Company/Standards/` |

## Manager Heuristics

| Signal | Green | Red |
|--------|-------|-----|
| Commits | Small, frequent, descriptive | 2+ days silent |
| Commit messages | Explain WHY | "stuff" / "wip" / "fix" |
| Session logs | Written daily | Missing or perfunctory |
| Research phase | Notes in vault before coding | Skipped straight to code |
| Shared/ contributions | Regular additions | Empty after 2+ weeks |
