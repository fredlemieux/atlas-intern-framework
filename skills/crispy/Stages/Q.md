# Stage Q — Questions

Generate focused research questions about the task. These go to the R stage — R should be able to answer them by reading the codebase alone.

## Rules
- Questions must be about *what exists*, never about *what to build*
- No implementation intent in question wording
- Cover: data models, file patterns, existing conventions, access patterns, service connections, edge cases, acceptance criteria
- Always include a question about *how the relevant parts of the system connect* — prevents missing key integration points

## Format

Write 5–8 numbered questions to `1-Q-questions.md`. Each question must be answerable by reading the codebase — no opinions, no guesses.

## Gate

Present questions. Wait for review and any additions before proceeding to R.
